Introduction à l'Ingénierie des Données

Vue d'ensemble

Dans ce module d'introduction à l'ingénierie des données, nous allons décrire le rôle d'un ingénieur des données et expliquer pourquoi l'ingénierie des données doit être réalisée dans le Cloud. Un ingénieur des données est une personne qui construit des pipelines de données. Nous commencerons donc par explorer ce que cela signifie, quels types de pipelines l'ingénieur des données construit et quel est leur objectif.

Nous examinerons les défis associés à l'ingénierie des données et comment beaucoup de ces défis sont plus faciles à résoudre lorsqu'on construit ses pipelines de données dans le Cloud. De plus, nous vous introduirons à BigQuery, le Data Warehouse serveur sans serveur de Google Cloud, capable de traiter des pétaoctets de données.

Lacs de Données et Entrepôts de Données

Après avoir défini ce que sont les lacs de données et les entrepôts de données, nous allons approfondir ces concepts. Les ingénieurs des données peuvent être responsables à la fois de :

  • Les systèmes de base de données transactionnels qui soutiennent les applications d'une entreprise.
  • Les entrepôts de données qui soutiennent les charges de travail analytiques.

Solutions Google Cloud

Dans cette leçon, nous allons explorer les différences entre les bases de données et les entrepôts de données, et examiner les solutions Google Cloud pour chacun de ces types de charges de travail. Puisque l'entrepôt de données sert également d'autres équipes, il est crucial d'apprendre à collaborer efficacement avec elles.

Gouvernance des Données et Sécurité

Dans le cadre d'une collaboration efficace, votre équipe d'ingénierie sera amenée à mettre en place des politiques d'accès aux données ainsi qu'une gouvernance globale concernant l'utilisation et la non-utilisation des données par vos utilisateurs. Nous discuterons de la manière d'offrir un accès à l'entrepôt de données tout en respectant les meilleures pratiques en matière de gouvernance des données.

Automatisation et Surveillance

Nous discuterons également de la manière de productionnaliser l'ensemble de l'opération, d'automatiser les processus et de surveiller autant que possible pour assurer une performance optimale.

Étude de Cas Pratique

Enfin, nous examinerons une étude de cas sur la manière dont un client de Google Cloud a résolu un problème commercial spécifique. Après cela, vous réaliserez un laboratoire pratique où vous utiliserez BigQuery pour analyser des données.
