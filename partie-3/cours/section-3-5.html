🔄 ELT ou ETL ? Bien choisir selon les cas

📌 Rappel sur ELT avec SQL

Dans la leçon précédente, nous avons vu comment utiliser SQL dans un pipeline ELT pour garantir la qualité des données. L’idée principale : vous n'avez pas toujours besoin d’un ETL. Même si des transformations sont nécessaires, l’approche ELT peut suffire.

⚠️ Mais ELT ne suffit pas toujours

Il existe cependant des situations où ELT atteint ses limites, notamment lorsque :

  • Les transformations ne peuvent pas être exprimées en SQL
  • Les transformations sont trop complexes pour du SQL pur

📚 Exemples concrets

  • 🌍 Traduction de contenu : Traduire un texte de l’espagnol vers l’anglais nécessite l’appel à une API externe (comme Cloud Translation API). Cela n’est pas faisable directement en SQL. On peut utiliser une fonction distante dans BigQuery, mais cela implique du développement en dehors de BigQuery.
  • 📈 Analyse comportementale : Suivre les actions de clients dans une fenêtre temporelle glissante peut être exprimé avec des agrégations SQL avancées, mais c’est bien plus simple via une logique programmée.

✅ Quand privilégier ETL ?

Voici les cas où un pipeline ETL devient indispensable :

  • 📋 Lorsque les données doivent être contrôlées, transformées ou enrichies avant d’être chargées dans BigQuery
  • 💡 Lorsque les transformations sont trop complexes pour du SQL
  • 🔁 Lorsque le chargement est en continu (streaming) – Dataflow le gère nativement
  • 🧪 Lorsque vous souhaitez intégrer avec des outils CI/CD et faire du test unitaire sur tous les composants

🏗️ Architecture recommandée (Google Cloud)

Voici le schéma typique d’un pipeline ETL recommandé :

  1. 📤 Extraire les données depuis Pub/Sub, Cloud Storage, Spanner, Cloud SQL, etc.
  2. 🔧 Transformer les données via Dataflow
  3. 📥 Charger les données dans BigQuery

🛠️ Les outils ETL de Google Cloud

Google Cloud propose plusieurs services pour l’ETL :

  • 🌀 Dataflow : Service entièrement géré et sans serveur basé sur Apache Beam. Il supporte le traitement batch et streaming. Des modèles prédéfinis permettent un démarrage rapide.
  • 🐘 Dataproc : Basé sur Apache Hadoop. Recommandé pour les utilisateurs expérimentés en Big Data. Permet l’exécution de jobs Spark, Hive ou MapReduce.
  • 🎛️ Data Fusion : Interface graphique simple pour créer des pipelines ETL sans écrire de code. Ces pipelines peuvent être déployés à grande échelle sur des clusters Dataproc.

Ces outils permettent de transformer les données avant de les stocker dans un lac de données ou un entrepôt de données, afin de faciliter des analyses avancées.
