Récapitulatif rapide de 🔹 EL, 🔸 ELT et 🔷 ETL

🔹 EL (Extraire et Charger) : Ce processus consiste à extraire et charger les données telles quelles dans un système.

  • 📤 Extraire : Extraire les données de leur source.
  • 📦 Charger : Charger directement les données brutes dans le système cible, comme un entrepôt de données (par exemple, BigQuery).

🔸 ELT (Extraire, Charger et Transformer) : Cela permet de charger les données brutes dans le système cible et de les transformer quand elles sont nécessaires. Par exemple, vous pouvez fournir un accès aux données brutes via une vue.

  • 📤 Extraire : Extraire les données depuis la source.
  • 📦 Charger : Charger les données dans le système cible.
  • 🔄 Transformer : Transformer les données à la demande via une vue ou un job SQL, selon les besoins.

🔷 ETL (Extraire, Transformer et Charger) : Ce processus d'intégration de données permet de transformer les données avant de les charger dans le système cible.

  • 📤 Extraire : Extraire les données de la source.
  • 🔄 Transformer : Transformer les données dans un service intermédiaire (par exemple, Dataflow).
  • 📦 Charger : Charger les données transformées dans le système cible, comme un entrepôt de données (par exemple, BigQuery).

Quand utiliser 🔹 EL ?

Utilisez EL uniquement si les données sont déjà propres et prêtes à être utilisées :

  • 📂 Fichiers journaux dans Cloud Storage (par exemple, CSV, JSON) peuvent être extraits et chargés directement dans BigQuery.
  • 🕹️ Micro-batchs : Chaque fois qu'un nouveau fichier arrive dans Cloud Storage, une Cloud Function se lance pour charger les données dans BigQuery.

Quand utiliser 🔸 ELT ?

ELT commence par un chargement de données, puis la transformation intervient après le chargement, selon les besoins :

  • 📂 Chargement direct : Les données brutes sont chargées dans le système cible, puis transformées via une vue ou une requête SQL selon les besoins des utilisateurs.
  • 🔄 Transformation dynamique : Vous stockez des données brutes, par exemple un fichier JSON, puis vous appliquez une transformation en fonction de la demande (par exemple, utilisation de l'API Vision et de BigQuery pour l'analyse du JSON).

Quand utiliser 🔷 ETL ?

Utilisez ETL quand vous devez transformer les données avant de les charger dans le système cible :

  • 🔄 Transformation préalable : Les données sont d'abord extraites et transformées dans un service comme Dataflow avant d'être chargées dans un entrepôt de données comme BigQuery.
  • 📦 Chargement après transformation : Les données sont nettoyées, filtrées et modifiées avant de rejoindre le stockage final.

🔄 Exemple de cas d'utilisation : Vous recevez une image, vous invoquez l'API Vision et récupérez des informations au format JSON. Vous stockez ce JSON dans son format brut dans BigQuery, puis vous l'analysez via une transformation SQL pour extraire des informations spécifiques, telles que des logos.
