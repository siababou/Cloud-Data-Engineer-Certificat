🎓 Comment utiliser des opérations dans BigQuery ?

Dans cette leçon, nous découvrons comment identifier et corriger les problèmes de qualité des données à l'aide des fonctionnalités SQL de BigQuery.

🔍 Types de Problèmes de Qualité et Solutions

  • 🧹 Filtrer les données invalides : Utiliser des Views avec une clause WHERE pour exclure par exemple les quantités négatives.
  • 📊 Groupes non significatifs : Utiliser GROUP BY suivi de HAVING pour exclure les groupes avec peu de données (< 10).
  • ⚠️ NULL vs BLANK :
    • NULL = absence de donnée
    • BLANK = chaîne vide ("")
    • 🎯 Penser à ce que vous souhaitez filtrer : l’un, l’autre ou les deux
  • 📈 Comptage de valeurs valides : Utiliser COUNTIF ou des conditions IF pour exclure certaines valeurs des calculs.

🎯 Exactitude

  • Vérifier les données par rapport à des valeurs de référence.
  • 💰 Exemple : recalculer le sub_total = quantity_ordered * item_price.
  • ✅ Utiliser IN pour valider si une donnée appartient à une liste canonique.

📉 Complétude

  • 📌 Identifier les données manquantes (NULL).
  • 🔄 Fonctions utiles : NULLIF, COUNTIF, COALESCE.
  • 🔗 Utiliser UNION pour intégrer des données manquantes (ex : mois absents).
  • ⚙️ Vérification des fichiers avec des checksums comme MD5.

🔁 Consistance

  • 📌 Problèmes fréquents : doublons, formats incohérents (ex : timezone, padding).
  • 🔍 Comparer COUNT vs COUNT DISTINCT pour détecter les doublons.
  • 🧽 Nettoyer les chaînes avec des fonctions de manipulation de texte.
  • 📏 Non-uniformité des unités (ex : cm vs mm) → utiliser CAST et FORMAT().
  • 📝 Toujours documenter les unités et les formats utilisés.

✅ En résumé, BigQuery offre des outils puissants pour traiter les problèmes de qualité sans nécessiter de transformation intermédiaire. Il est essentiel d’utiliser judicieusement les fonctions SQL pour garantir la validité, l’exactitude, la complétude, la consistance et l’uniformité des données.
